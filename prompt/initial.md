# [기획서] AI Agent 기반 PRD 생성 서비스

## 1. 개요

- **목적**: 사용자의 입력을 분석하여 프로젝트 생성에 필요한 PRD(Product Requirements Document)를 자동 생성하는 서비스
- **타겟**: 아이디어를 빠르게 문서화하고자 하는 기획자, 개발자, 1인 창업자

---

## 2. 화면 구성 및 프로세스 (Main Page)

메인 화면은 크게 3단계의 질문 과정을 거치며, 사용자의 입력을 인식하여 순차적으로 질문이 활성화됩니다.
모바일 / 데스크톱을 지원합니다.

### 2.1 1단계: 서비스 개요 질문

- **Q1. 서비스 명칭 및 정의**: 서비스 이름과 핵심 문제 해결 대상(누가, 무엇을) 명시
- **Q2. 핵심 기능(Top 3)**: 우선순위에 따른 필수 기능 3가지
- **Q3. 주요 화면 구성**: 메인/마이페이지 외 특화 화면(예: 대시보드, 채팅창 등)
- **Q4. 사용자 플로우(User Journey)**: 앱 진입부터 최종 목적 달성까지의 단계
- **Q5. 톤앤매너**: 서비스의 말투 및 안내 메시지 스타일 (예: 친절한 가이드형, 전문적인 스타일)

### 2.2 2단계: 디자인 요소 질문

_질문에 직접 답하지 않고 하단 버튼을 통해 테마를 중복 선택할 수 있습니다._

- **테마 옵션**: 미니멀, 화려한 인터랙션, 신뢰감 있는, 귀여운 등
- **Q1. 브랜드 키워드**: 분위기를 나타내는 키워드 3가지
- **Q2. 컬러 시스템**: 포인트 컬러 및 배경색(다크모드 지원 여부)
- **Q3. 타이포그래피**: 고딕(가독성) vs 명조(감성) 또는 특정 폰트 지정
- **Q4. UI 디테일**: 버튼 곡률(Round), 아이콘 굵기, 그림자 효과 수준
- **Q5. 레퍼런스**: 벤치마킹할 서비스나 디자인 스타일

### 2.3 3단계: 기술 제약 질문

- **Q1. 선호 기술 스택**: Frontend(React, Next.js 등) 및 Backend(Node.js 등)
- **Q2. 데이터 관리**: 실시간 처리 필요성 및 대용량 미디어 처리 여부
- **Q3. 외부 API 연동**: 결제(PortOne), 지도(Google), 소셜 로그인 등
- **Q4. 보안 및 인증**: 단순 이메일 가입 vs 2단계/생체 인증
- **Q5. 예외 상황(Edge Cases)**: 네트워크 장애, 데이터 부재 시 대응 정책

## 3. 주요 기능 및 컴포넌트

### 3.1 버튼 구성

- **PRD 생성 버튼**: 화면에 항상 고정(Floating). 미입력 항목 존재 시 "이 상태로 생성하시겠습니까?" Alert 노출.
- **초기화 버튼**: 모든 입력 데이터를 초기화하고 첫 단계로 이동.
- **로그인 버튼**: Google OAuth 연동. (로그인 시 과거 생성 PRD 보기 및 데이터 불러오기 가능)

### 3.2 PRD 생성 모달

- 입력된 최종 내용을 확인하는 단계.
- '생성하기' 버튼 클릭 시 Gemini API를 호출하여 문서 생성 시작.

## 4. 기술 사양 (Tech Stack)

| 구분           | 기술 스택                                  |
| :------------- | :----------------------------------------- |
| **Frontend**   | Next.js (App Router), TypeScript           |
| **Deployment** | Vercel                                     |
| **Database**   | Supabase (PostgreSQL)                      |
| **Auth**       | Google OAuth (via Supabase)                |
| **AI Engine**  | Google Gemini API, Vercel AI SDK           |
| **Rendering**  | 첫 페이지 SSG(Static Site Generation) 적용 |

---

## 5. 개발 및 운영 가이드라인

- **디자인 원칙**: 미니멀한 UI, 세련된 인터랙션 지향. 전반적으로 lovable과 bolt의 UI를 참고할 것.
- **프로그래밍 패러다임**: 함수형 프로그래밍과 OOP(객체지향) 원칙을 최우선으로 고려.
- **결과물 제공**: 생성된 PRD는 `.md`(마크다운) 또는 `.pdf` 파일로 다운로드 가능.
- **접근성**: PRD 생성 자체에 로그인을 필수로 강제하지 않음 (비로그인 사용자 허용).
- **SEO**: 검색 엔진 최적화를 위해 첫 페이지를 정적 생성으로 처리.
